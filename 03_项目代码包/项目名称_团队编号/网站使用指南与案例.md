# 网站使用指南与案例

项目名称：钢材表面缺陷智能质检与大模型诊断报告系统

适用对象：课程展示、功能测试、答辩演示、后续录制演示视频。

## 1. 启动网站

进入代码包目录：

```powershell
cd 03_项目代码包\项目名称_团队编号
```

安装依赖后启动：

```powershell
python -m streamlit run app.py
```

浏览器打开：

```text
http://localhost:8501
```

Windows 下也可以双击 `run.bat` 启动。

双击启动后会出现一个命令行窗口。该窗口就是本地网站服务进程，使用网页期间不要关闭；如果窗口提示 8501 端口已有服务运行，脚本会直接打开已有页面。

## 2. 页面结构

| 页面 | 用途 |
| --- | --- |
| 单图检测 | 上传或选择一张钢材表面图片，输出缺陷类别、置信度、Top-K 结果、风险等级和处理建议 |
| 批量检测 | 对多张图片进行快速复核，输出明细表、类别分布、平均置信度、高风险样本数，并支持 CSV 和 Markdown 下载 |
| 缺陷知识库 | 展示六类钢材表面缺陷的中文解释、典型成因和处理建议 |
| 模型结果 | 展示测试准确率、宏平均 F1、混淆矩阵、准确率曲线和损失曲线 |
| 典型案例 | 提供生产线抽检、异常批次复核、质检报告辅助三个演示案例 |
| 使用指南 | 在网页内查看操作步骤、演示顺序和常见问题 |

## 3. 单图检测流程

1. 在左侧栏选择一个测试样例，或上传 `jpg/jpeg/png/bmp` 图片。
2. 如需使用大模型增强报告，勾选“单图检测时调用 My Codex”。
3. 点击“开始检测”。
4. 查看预测缺陷、置信度、推理耗时、Top-K 结果和质检解释。
5. 点击“下载本次质检报告 Markdown”保存单图报告。

说明：My Codex 接口只在单图检测中按需调用。接口不可用时，系统仍会使用本地缺陷知识库生成稳定报告。

公开仓库中已保留 `data/samples/` 下的 6 张轻量示例图，因此即使没有下载完整 NEU-CLS 原始数据集，也可以先运行页面查看单图检测和案例演示。

## 4. 批量检测流程

1. 打开“批量检测”页面。
2. 选择“测试集前 30 张”或“上传多张图片”。
3. 点击“运行批量检测”。
4. 查看检测明细、预测类别分布、平均置信度和高风险样本数。
5. 下载 `batch_detection_results.csv` 和 `batch_detection_summary.md`。

批量检测默认不逐张调用 My Codex，目的是保证速度稳定，并避免接口等待时间过长。

## 5. 推荐课堂演示顺序

1. 打开网站首页，展示测试集准确率、宏平均 F1、测试样本数和 My Codex 状态。
2. 在“单图检测”中选择一个内置样例，运行检测并展示报告下载。
3. 打开“模型结果”，展示混淆矩阵、训练曲线和关键指标。
4. 打开“批量检测”，运行测试集样本批量复核，展示 CSV 和 Markdown 摘要下载。
5. 打开“典型案例”，按案例说明系统在工程质检场景中的应用价值。

## 6. 典型案例

### 案例一：生产线龟裂抽检

场景：热轧或冷轧产线抽检发现疑似裂纹纹理，需要快速判断是否扩大抽检。

操作：

1. 打开“典型案例”页面。
2. 找到“生产线龟裂抽检”。
3. 点击“在单图检测中打开”。
4. 勾选 My Codex。
5. 运行检测并下载 Markdown 报告。

讲解重点：如果系统识别为龟裂且置信度较高，应强调龟裂属于高风险表面缺陷，需要重点复检，并追溯轧制温度、张力控制和冷却过程记录。

### 案例二：异常批次快速复核

场景：同一批次多张图片存在氧化皮压入疑似缺陷，需要快速统计缺陷分布。

操作：

1. 打开“批量检测”页面。
2. 选择“测试集前 30 张”。
3. 点击“运行批量检测”。
4. 查看预测类别分布和高风险样本数。
5. 下载 CSV 明细和 Markdown 摘要。

讲解重点：批量结果适合用于异常批次复核。若某类缺陷集中出现，可追溯除鳞水压、喷嘴状态、加热炉温度和停留时间等工艺参数。

### 案例三：质检报告辅助撰写

场景：外观要求较高的产品出现划痕，需要形成给质量人员或工艺人员阅读的文字说明。

操作：

1. 打开“典型案例”页面。
2. 找到“质检报告辅助撰写”。
3. 点击“在单图检测中打开”。
4. 开启 My Codex。
5. 运行检测，查看大模型生成报告并下载 Markdown。

讲解重点：该案例展示大模型不直接替代模型分类，而是把分类结果、置信度、缺陷知识库和处理建议组织成更自然的质检报告。

## 7. 常见问题

| 问题 | 处理方式 |
| --- | --- |
| 页面提示未找到模型权重 | 运行 `python models\train_neu_cls.py --epochs 8`，或确认 `models/artifacts/neu_cls_cnn/best_model.pt` 是否存在 |
| My Codex 不可用 | 检查本机 `%USERPROFILE%\.codex\config.toml` 和 `%USERPROFILE%\.codex\auth.json`，或直接使用本地规则报告 |
| 上传图片无法识别 | 确认图片格式为 `jpg/jpeg/png/bmp`，并尽量使用钢材表面缺陷图像 |
| 批量检测较慢 | 减少样本数量，或优先使用测试集内置样例验证流程 |
| 中文显示异常 | 确认文件按 UTF-8 保存，命令行乱码通常不影响 Streamlit 页面显示 |

## 8. 可用于报告的增强点

- 系统不只做单张图片分类，还支持批量检测与结果导出。
- 系统将深度学习分类结果与缺陷知识库、大模型报告生成结合，形成“识别 + 解释 + 建议”的闭环。
- 网站内置典型案例和使用指南，方便课堂展示和后续视频录制。
- 批量检测摘要可以直接作为功能测试材料，支持结果留痕。
