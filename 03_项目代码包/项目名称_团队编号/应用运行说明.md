# 应用运行说明

本项目已完成钢材表面缺陷检测、模型推理、批量复核、质检解释和 My Codex 增强报告模块整合。

## 一键运行

Windows 下双击：

```text
run.bat
```

脚本会启动 Streamlit 页面。

## 命令行运行

```powershell
python -m streamlit run app.py
```

启动后在浏览器打开：

```text
http://localhost:8501
```

## 网站页面

| 页面 | 功能 |
| --- | --- |
| 单图检测 | 上传或选择样例图片，输出缺陷类别、置信度、推理耗时、Top-K 结果和质检报告 |
| 批量检测 | 对测试集样本或多张上传图片进行批量推理，导出 CSV 明细和 Markdown 摘要 |
| 缺陷知识库 | 查看六类缺陷的中文解释、典型成因和处理建议 |
| 模型结果 | 查看准确率、宏平均 F1、混淆矩阵、准确率曲线和损失曲线 |
| 典型案例 | 查看生产线抽检、异常批次复核、质检报告辅助三个演示案例 |
| 使用指南 | 查看网站内置操作步骤、演示顺序和常见问题 |

## 大模型接口配置

默认使用本地规则知识库生成稳定报告，不需要密钥。

应用会优先读取本机 Codex 配置：

```text
%USERPROFILE%\.codex\config.toml
%USERPROFILE%\.codex\auth.json
```

当前已验证的 My Codex 配置为：

```text
base_url = "https://api.9e.lv/v1"
wire_api = "responses"
model = "gpt-5.5"
```

如果需要手动覆盖 Codex API，可配置：

```powershell
$env:CODEX_API_URL="https://api.9e.lv/v1"
$env:CODEX_MODEL="gpt-5.5"
$env:CODEX_WIRE_API="responses"
```

如本地接口需要密钥，再配置：

```powershell
$env:CODEX_API_KEY="your_key"
```

不要将真实密钥写入代码、报告或提交材料。
